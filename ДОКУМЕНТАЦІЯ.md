## 1. Загальний опис системи

### 1.1 Призначення

Вебсистема підтримки колективного прийняття рішень (КПРС) призначена для організації та автоматизації процесу групового експертного оцінювання альтернатив. Система дозволяє:

- створювати сесії експертного оцінювання з довільним набором альтернатив та експертів;
- збирати індивідуальні оцінки кожного експерта в кількох раундах;
- агрегувати думки експертів за п'ятьма різними методами КПРС;
- вимірювати рівень консенсусу між експертами за допомогою статистичних метрик;
- формувати PDF-звіт за результатами сесії.

### 1.2 Область застосування

Система може використовуватися в академічних, управлінських та дослідницьких контекстах, де потрібно:

- провести порівняльний аналіз альтернатив із залученням кількох незалежних фахівців;
- забезпечити прозорість та обґрунтованість колективного рішення;
- дослідити вплив методу агрегування на підсумковий рейтинг альтернатив.

### 1.3 Технічні обмеження

- Система є демонстраційним прототипом і не розрахована на виробниче (production) навантаження.
- Відсутня автентифікація користувачів.
- Секретний ключ Flask захардкоджено (`"demo-secret-key"`).
- Усі маршрути загальнодоступні.

| Поле | Тип | Обмеження | Опис |
|------|-----|-----------|------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | Унікальний ідентифікатор оцінки |

## 5. Методи прийняття рішень

Усі методи реалізовано в [services/methods.py](services/methods.py). Вхідні параметри:
- `latest_scores` — словник `{expert_id: {alt_name: score}}` для останнього раунду;
- `expert_data` — список `[{"id": ..., "name": ..., "weight": ...}]`;
- `alternatives` — список назв альтернатив.

### 5.1 Зважений середній бал (`weighted_score_method`)

**Ідея:** кожна альтернатива отримує зважену середню оцінку за всіма експертами.

**Формула:**

```
S(a) = Σ(i=1..m) [w_i × s_i(a)] / Σ(i=1..m) w_i
```

де:
- `s_i(a)` — оцінка i-го експерта для альтернативи a;
- `w_i` — коефіцієнт компетентності i-го експерта;
- `m` — кількість експертів.

**Переможець:** альтернатива з найбільшим значенням `S(a)`.

---

### 5.2 Метод Борда (`borda_method`)

**Ідея:** кожен експерт ранжує альтернативи; альтернативі на k-й позиції нараховується `(n − k − 1)` балів Борда (зважених на компетентність).

**Формула:**

```
B(a) = Σ(i=1..m) w_i × (n − r_i(a) − 1)
```

де:
- `r_i(a)` — ранг альтернативи a у списку i-го експерта (0 = найкраща);
- `n` — кількість альтернатив.

Ранжування кожного експерта будується за спаданням оцінки (функція `_ranking_for_expert`).

**Переможець:** альтернатива з максимальним `B(a)`.

---

### 5.3 Метод Кондорсе (`condorcet_method`)

**Ідея:** попарне порівняння кожної пари альтернатив. Альтернатива, яку більшість (зважена) експертів оцінила вище суперника, отримує 1 перемогу; при рівності — по 0.5 кожній.

**Алгоритм:**

Для кожної пари `(a, b)`:
```
score_a = Σ w_i  (для всіх i, де s_i(a) > s_i(b))
score_b = Σ w_i  (для всіх i, де s_i(b) > s_i(a))

якщо score_a > score_b → wins[a] += 1
якщо score_b > score_a → wins[b] += 1
інакше               → wins[a] += 0.5, wins[b] += 0.5
```

**Переможець:** альтернатива з найбільшою кількістю перемог `wins[a]`.

---

### 5.4 Метод Делфі (`delphi_method`)

**Ідея:** імітує ітеративну процедуру Делфі — поточна оцінка корегується з урахуванням попереднього раунду (вага 70% / 30%).

**Формула:**

```
D(a) = Σ(i=1..m) w_i × (0.7 × s_i_cur(a) + 0.3 × s_i_prev(a)) / Σ(i=1..m) w_i
```

де:
- `s_i_cur(a)` — оцінка i-го експерта в останньому раунді;
- `s_i_prev(a)` — оцінка i-го експерта в передостанньому раунді (якщо раунд один, використовується той самий).

**Переможець:** альтернатива з максимальним `D(a)`.

---

### 5.5 Метод АНР (метод парних порівнянь) (`ahp_like_method`)

**Ідея:** спрощена реалізація методу аналізу ієрархій (АНР / AHP). На основі зважених середніх балів будується матриця попарних порівнянь у шкалі 1–9, потім розраховується вектор пріоритетів.

**Алгоритм:**

1. Обчислити зважені бали `weighted[a]` методом зваженого середнього.
2. Побудувати матрицю `M[i][j] = scale(weighted[a_i] / weighted[a_j])`, де `scale` обмежує відношення діапазоном [1/9, 9]:
   ```
   scale(ratio) = min(9, max(1, ratio))       якщо ratio ≥ 1
               = 1 / min(9, max(1, 1/ratio))  якщо ratio < 1
   ```
3. Нормалізувати кожен стовпець: `M_norm[i][j] = M[i][j] / Σ_k M[k][j]`.
4. Вектор пріоритетів: `p(a_i) = Σ_j M_norm[i][j] / n`.

**Переможець:** альтернатива з максимальним `p(a)`.

## 6. Метрики консенсусу

Реалізовано в [services/metrics.py](services/metrics.py).

### 6.1 Коефіцієнт конкордації Кендалла W (`kendall_w_from_rankings`)

**Призначення:** вимірює ступінь узгодженості рейтингів між усіма експертами. Значення W ∈ [0; 1], де:
- `W = 0` — повна відсутність узгодженості;
- `W = 1` — повна узгодженість.

**Формула:**

```
W = 12S / (m² × (n³ − n))
```

де:
- `S = Σ(a) (R(a) − R̄)²` — сума квадратів відхилень сум рангів від середнього;
- `R(a) = Σ(i) r_i(a)` — сума рангів альтернативи a по всіх експертах;
- `R̄ = Σ R(a) / n` — середня сума рангів;
- `m` — кількість експертів;
- `n` — кількість альтернатив.

---

### 6.2 Коефіцієнт варіації (`variation_coefficient`)

**Призначення:** для кожної альтернативи вимірює відносний розкид оцінок між експертами.

**Формула:**

```
CV(a) = σ(a) / μ(a)
```

де:
- `σ(a)` — генеральне стандартне відхилення оцінок для альтернативи a;
- `μ(a)` — середнє значення оцінок для альтернативи a.

Чим менший CV, тим вища узгодженість думок щодо даної альтернативи.

---

### 6.3 Ентропія (`entropy_metric`)

**Призначення:** для кожної альтернативи вимірює невизначеність або «розпорошеність» оцінок.

**Формула:**

```
H(a) = −Σ(j) p_j × log₂(p_j)
```

де:
- `p_j = s_j(a) / Σ_k s_k(a)` — нормалізована оцінка j-го експерта для альтернативи a.

Чим менша H(a), тим більша узгодженість у оцінках цієї альтернативи.

## 7. Генерація PDF-звіту

Маршрут `GET /report/<id>` формує PDF-документ у пам'яті (без запису на диск) за допомогою бібліотеки **ReportLab** та повертає його як вкладення (`Content-Disposition: attachment`).

**Вміст звіту:**

1. Заголовок «Звіт сесії експертного оцінювання».
2. Назва та опис сесії.
3. Коефіцієнт конкордації Кендалла W.
4. Для кожного з 5 методів: назва методу, бали по альтернативах, рекомендована альтернатива.
5. Додаткові показники: CV та H для кожної альтернативи.

**Технічні деталі:**
- Розмір сторінки: A4.
- Формат файлу: `session_<id>_report.pdf`.
- Шрифт: Helvetica (вбудований у ReportLab).
- Автоматичний перенос на нову сторінку при переповненні (`y < 100`).

## 9. Інструкція користувача

### 9.1 Перегляд існуючих сесій

На головній сторінці (`/`) відображається список усіх сесій. Натисніть на назву сесії для перегляду результатів.

### 9.2 Створення нової сесії

1. Натисніть «Нова сесія» у шапці.
2. Заповніть форму:
   - **Назва** — коротка назва задачі прийняття рішень.
   - **Опис** — детальний опис контексту.
   - **Альтернативи** — введіть назви альтернатив, кожну з нового рядка (мінімум 2).
   - **Експерти** — введіть імена, кожне з нового рядка (мінімум 2).
   - **Коефіцієнти компетентності** — числа від 0 до 1 для кожного експерта в тому ж порядку (наприклад: `0.9`).
3. Натисніть «Створити».

### 9.3 Введення оцінок

На сторінці сесії відображається таблиця для введення оцінок:
1. Виберіть номер раунду.
2. Для кожного рядка (експерт) введіть числові оцінки для кожної альтернативи.
3. Натисніть «Зберегти оцінки».

Система автоматично перерахує результати всіх методів та метрики.

### 9.4 Перегляд результатів

На сторінці сесії після збереження оцінок відображаються:
- **Результати 5 методів** — рейтинги та рекомендована альтернатива.
- **Коефіцієнт конкордації Кендалла W** — загальна узгодженість.
- **Коефіцієнт варіації (CV)** та **Ентропія (H)** — по кожній альтернативі.
